Self‑hosted bias simulation with signed evidence and regulator‑ready reports.

Architecture

  • Multi-container via docker-compose (API, Worker, Redis).
  • Single-tenant AMI via AWS Marketplace.
  • Optional Prometheus; /metrics endpoint exposed for monitoring.

Dual-branch calibration

  • Amplification branch: trained on features + historic decisions.
  • Intrinsic branch: trained on features only to surface achievable fairness.
  • Outputs show the delta — e.g., historical DI vs. achievable DI.

Evidence bundle

30-page PDF with methodology, signed manifest (dataset hash, RNG seed, software version), and certificate pack covering data quality, model fidelity, and regulatory mapping.

Deployment boundary

Runs inside your VPC. Default configuration has zero outbound calls. Optional usage metrics and Prometheus endpoints are opt-in per customer policy.

How it Works

1) Data ingestion

Upload structured data or summary statistics. FL-BSA establishes amplification vs. intrinsic baselines.

2) Simulation

Synthesize borrower records with embedded decisions. Evaluate AIF360/Fairlearn metrics (DI, EO, TPR gaps).

3) Evidence

Export the signed manifest, PDF, and certificates. Artifacts chain task ID → model seeds → outputs for auditability.

4) Review

Share the evidence bundle with Compliance, Model Risk, and Internal Audit. Optional advisory runbooks available.

Deployment options

AWS AMI

Single-tenant AMI. Boots deterministic services via systemd and exposes API on 8080 (configurable).

Container package

docker-compose stack (API, Worker, Redis). CPU-only and GPU-preferred profiles validated at scale.

In practice

Example: A bank ingests 250k historic applications including credit decisions. FL-BSA reports a historical disparate impact (DI) of 0.68 and an achievable DI of 0.83 after isolating decision leakage—documented with certificate chain and signed manifest.

Illustrative scenario only; not a customer claim.
