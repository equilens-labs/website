Color System

The FL-BSA color system uses Indigo as the primary brand color, with a comprehensive neutral palette for text and surfaces.

Primary Palette

  • Primary: --color-primary / #4f46e5
  • Primary Hover: --color-primary-hover / #4338ca
  • Primary Light: --color-primary-light / #6366f1
  • Primary Dark: --color-primary-dark / #3730a3

Neutral Palette

  • White: --color-white / #ffffff
  • Gray 50: --color-gray-50 / #f9fafb
  • Gray 200: --color-gray-200 / #e5e7eb
  • Gray 500: --color-gray-500 / #6b7280
  • Gray 900: --color-gray-900 / #111827

Semantic Colors

  • Success: --color-success / #10b981
  • Warning: --color-warning / #f59e0b
  • Error: --color-error / #ef4444
  • Info: --color-info / #3b82f6

CSS Variables

:root {
  /* Primary */
  --color-primary: #4f46e5;
  --color-primary-hover: #4338ca;

  /* Text */
  --text-primary: #111827;
  --text-secondary: #374151;
  --text-muted: #4b5563;

  /* Backgrounds */
  --bg-main: #ffffff;
  --bg-subtle: #f8fafc;

  /* Borders */
  --border-default: #e5e7eb;
}

Typography

The FL-BSA design system uses system fonts for optimal performance and readability.

Font Stack

font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto,
             'Helvetica Neue', Arial, sans-serif;

Type Scale

Heading 1 — clamp(2rem, 5vw, 3rem)

Heading 2 — clamp(1.5rem, 3vw, 2rem)

Heading 3 — clamp(1.25rem, 2vw, 1.5rem)

Heading 4 — 1.25rem (20px)

Body text — 1rem (16px) with 1.5 line height for optimal readability in long-form content.

Lead text — clamp(16px, 1.6vw, 18px)

Note text — 0.95rem, italic, muted color

Buttons

All button variants meet WCAG AA contrast requirements and have 44px minimum touch targets.

Button Variants

Buttons use the `.btn` base class with `.btn-primary`, `.btn-secondary`, and `.btn-ghost` modifiers.

Button States

  • Default: Clear, high-contrast colors
  • Hover: Subtle translateY(-1px) with enhanced shadow
  • Active: Returns to baseline position
  • Focus: 2px outline with 2px offset for keyboard navigation
  • Disabled: 60% opacity, no pointer events

Forms

Form Inputs

Form elements follow the base typography and spacing rules defined in base.css. Labels, inputs, selects, and textareas inherit the system font stack and color tokens.

Form Guidelines

  • All inputs have 1px border that changes to primary color on focus
  • Focus states include a 3px shadow ring for clear indication
  • Labels use semibold font weight for clarity
  • Hover states provide immediate visual feedback

FAQ / Accordion

Custom-styled disclosure widgets with expand/collapse icons.

What is the FL-BSA design system?

The FL-BSA design system is a comprehensive collection of design tokens, components, and guidelines that ensure consistency across all FL-BSA product pages.

How do I use these components?

Simply reference the base.css stylesheet and apply the documented class names to your HTML elements. All components are fully responsive and accessible.

Features

  • Custom expand/collapse icon (+ becomes − when open)
  • Smooth transitions and hover states
  • Background color change when open
  • Border color change to primary when active
  • Fully keyboard accessible

Accessibility

Focus States

All interactive elements have visible focus indicators:

  • 2px outline with 2px offset using :focus-visible
  • Primary color outline for consistency
  • Respects user's prefers-reduced-motion preference

Color Contrast

All text/background combinations meet WCAG AA standards:

  • Body text: 4.5:1 minimum contrast ratio
  • Large text (18px+): 3:1 minimum contrast ratio
  • Interactive elements: Clear visual differentiation
  • Hover/focus states: Enhanced visibility

Keyboard Navigation

  • Skip-to-content link for quick navigation
  • Logical tab order throughout
  • All interactive elements keyboard accessible
  • ARIA labels where appropriate

Screen Readers

<!-- Screen reader only text -->
<span class="sr-only">Accessible description</span>

<!-- Skip to content link -->
<a class="skip-to-content" href="#main">
  Skip to content
</a>

Reduced Motion

The design system respects the user's motion preferences:

@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}

Touch Targets

  • All interactive elements: minimum 44×44px
  • Mobile navigation: optimized for thumb reach
  • Adequate spacing between clickable elements
