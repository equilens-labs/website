Deployment steps

1

Request guided pilot access

AWS Marketplace listing coming soon. Contact Equilens to request guided pilot access and pre-release deployment materials.

2

Prepare your environment

We will confirm the approved deployment path for your environment. Select instance type, region, VPC, and subnet per your change controls.

3

Boot the appliance

Boot runs docker-compose via systemd. The appliance listens on port 8080 (configurable), with optional Prometheus on 9090.

4

Run a test

Upload sample data or a sandbox dataset. Produce the PDF report, evidence manifest (SHA-256), and checksum log.

5

Confirm pilot scope

If evaluation fit is confirmed, we can align pilot terms or a Private Offer for enterprise rollouts.

Procurement package

Evidence bundle

  • Evidence manifest (JSON) and PDF report with embedded hashes.
  • Infrastructure BOM and network diagram (request access).
  • Control mapping aligned to FCA, EU AI Act, and ECOA checkpoints.

Current public demo materials (synthetic/demo-only, prerelease v5.0.0-rc4): customer_report.pdf, manifest.json, SHA256SUMS.txt.

Commercials

  • Commercial terms are finalized during pilot qualification; annual, multi-year, and Marketplace structures depend on the release path.
  • Pilot evaluations are scoped with agreed duration and success criteria.
  • Private Offer support for custom terms or multi-entity rollouts.

Deployment options

Container (VM)

docker-compose stack (API, Worker, Redis). CPU-only profile available; GPU preferred for faster synthetic generation.

Alternate: AWS AMI

Single-tenant AMI with deterministic boot. Delivers API, worker, and reporting services; integrates with customer IAM.

Coordinate CSP, HSTS, and CAA updates through your network and DNS teams. We provide implementation runbooks on request.
